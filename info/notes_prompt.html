Making Handwritten Notes Accessible HTML

This prompt was developed by Sean Maley, intended for math faculty who rely on handwritten demonstrations and notes. You can make a copy and edit the prompt to whatever your use case might be.

  1. Use the + button to add the file you wish to convert.
    Also select "Pro/Thinking" and "Canvas" under Tools.
  2. Copy and use this "Master Prompt" in the chat:
    3. <pre><code> "Here is a PDF file of handwritten math notes. Please convert it into a single, accessible, and downloadable HTML file for my visually impaired students. When you convert it, you must follow these specific rules: 1. Semantic HTML: Transcribe all text into structured HTML. Use &lt;h1&gt; for the main title, &lt;h2&gt; and &lt;h3&gt; for sections/sub-sections, &lt;p&gt; for paragraphs, and &lt;ul&gt;/&lt;ol&gt; for lists. 2. LaTeX for Math: Transcribe all handwritten mathematical notation directly into LaTeX syntax. Use \( ... \) for inline math (math that appears within a line of text). Use \[ ... \] for display math (math that is centered on its own line). 3. MathJax Configuration: In the &lt;head&gt; of the HTML file, you must include the MathJax library, configured to find and render the TeX (LaTeX) syntax. The script block must be this exact one: HTML &lt;!-- MathJax Configuration --&gt; &lt;script&gt; MathJax = { tex: { inlineMath: [['\\(', '\\)']], displayMath: [['\\[', '\\]']] } }; &lt;/script&gt; &lt;script id="MathJax-script" async src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-chtml.js"&gt;&lt;/script&gt; 4. Styling (Mandatory): In the &lt;head&gt; of the HTML file, you must include a &lt;style&gt; tag containing this exact CSS: &lt;style&gt; @import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap'); :root { --color-bg: #fdfdfd; --color-text: #1f2937; --color-primary: #10b981; /* Tailwind Emerald 500 */ --color-accent: #3b82f6; /* Tailwind Blue 500 */ } body { font-family: 'Inter', sans-serif; background-color: var(--color-bg); color: var(--color-text); line-height: 1.6; margin: 0; padding: 0; display: flex; justify-content: center; } .container { width: 100%; max-width: 900px; padding: 2rem; } h1 { font-size: 2.5rem; color: var(--color-primary); border-bottom: 4px solid var(--color-primary); padding-bottom: 0.5rem; margin-top: 0; margin-bottom: 1.5rem; font-weight: 700; } h2 { font-size: 1.75rem; color: var(--color-accent); margin-top: 2rem; margin-bottom: 1rem; font-weight: 600; } h3 { font-size: 1.3rem; color: var(--color-text); margin-top: 1.5rem; margin-bottom: 0.75rem; font-weight: 600; } p, ul, ol, pre { margin-bottom: 1rem; } ul, ol { padding-left: 1.5rem; } li { margin-bottom: 0.5rem; } .math-block { padding: 1rem; background-color: #f3f4f6; border-left: 5px solid var(--color-accent); margin: 1.5rem 0; overflow-x: auto; } .note { font-style: italic; color: #4b5563; margin-top: 0.5rem; display: block; } &lt;/style&gt; 5. Structure: Wrap all the main content (after &lt;body&gt;) in a div with the class container to apply the boxed styling. The final result should be a single, self-contained HTML file. This process will ensure the math is both easy to author and fully accessible."</code></pre>
